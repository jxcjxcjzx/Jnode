 title : The NextLevel Open Source Project on Open Hub
 summary : <p>Working on a large datawarehouse project has intrigued me about reverse engineering Relational Database Management Systems (RDBMS) to the point that I had to sit down, invent, and develop the NextLevel tool. The aim of this tool is to compute dependencies between tables, views, procedures, and triggers in a database. This information may seem trivial to get at, however, things get complicated given a few hundred triggers, procedures, views, and of course tables.</p>

<p>NextLevel may be a useful tool for:</p>

<p>* Reverse-engineering database objects (Procedures, Triggers, Views, Tables),</p>

<p>* Debugging your stored procedures and trigger definitions,</p>

<p>* Finding obsolete database objects,
<br/>* Finding access points from a given database to another database,
<br/>* documenting database objects and their usage,
<br/> * and other tasks I have not been thinking about, this far ;)</p>
